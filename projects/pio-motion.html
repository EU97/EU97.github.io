PIOMotion

Inertial Motion Unit asynchronous reading — high-frequency IMU data acquisition for UAV and robotics

View on GitHub

Overview

PIOMotion provides asynchronous firmware for reading Inertial Measurement Unit (IMU) sensors with minimal latency. By leveraging the Programmable I/O (PIO) state machines on microcontrollers such as the Raspberry Pi Pico, it achieves deterministic, high-frequency sensor reading independent of the main CPU execution flow.

This is essential for UAV flight controllers and robotics applications where attitude data must be acquired at consistent high rates without blocking other real-time tasks.

Key Features

  • Asynchronous IMU reading using PIO state machines (RP2040)
  • High-frequency attitude data acquisition (accelerometer + gyroscope)
  • Non-blocking design compatible with real-time control loops
  • I2C/SPI interface support for popular IMU chips (MPU-6050, ICM-42688)
  • Data buffering and DMA integration for efficient CPU offload
  • Plug-and-play integration with UAV flight controller pipelines

Technical Context

Developed during UAV platform integration work for the ONR Global research project. High-frequency, reliable IMU data is fundamental to the attitude estimation and control loops in multi-rotor systems. PIOMotion ensures consistent sensor sampling rates that satisfy the Nyquist requirements for state estimation filters (complementary, Kalman).
